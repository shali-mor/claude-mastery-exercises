# /explain

Explain $ARGUMENTS in plain English suitable for a junior developer.

1. **What it does** — one-sentence summary
2. **How it works** — step-by-step breakdown
3. **Why it exists** — its purpose in the broader codebase
4. **Common pitfalls** — mistakes or misunderstandings to avoid

Keep explanations concise. Use bullet points. Avoid jargon where possible.
