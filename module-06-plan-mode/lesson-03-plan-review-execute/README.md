# Lesson 03: Plan → Review → Execute

Practice the full Plan Mode cycle on a realistic refactoring task.

## What you'll build
- A plan document for a multi-file refactoring
- A review checklist to validate a plan before executing
- An execution log tracking what actually changed vs the plan

## Key concepts
- A good plan lists: affected files, order of changes, rollback steps, test strategy
- Review the plan for: missing dependencies, untested edge cases, wrong assumptions
- During execution: Claude reads the plan and follows it step by step
- After execution: compare what happened vs the plan — update for next time
