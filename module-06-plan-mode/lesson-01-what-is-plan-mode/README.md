# Lesson 01: What Is Plan Mode?

Understand Plan Mode's read-only constraint and when it saves you from costly mistakes.

## What you'll build
- A scenario analysis: plan first vs dive in
- A CLAUDE.md that activates Plan Mode for complex tasks
- A worksheet of the 5-phase planning workflow

## Key concepts
- Plan Mode = read-only: Claude can browse code but cannot edit files or run commands
- Activate with `Shift+Tab` or ask Claude to "think through this before doing anything"
- Exit with `ExitPlanMode` tool or `Shift+Tab` again
- Use `AskUserQuestion` inside Plan Mode to clarify before committing
- Rule of thumb: if a task touches > 3 files or has irreversible steps, plan first
