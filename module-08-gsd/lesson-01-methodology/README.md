# Lesson 01: GSD Methodology

Understand the GSD (Get Stuff Done) framework's Milestone → Phase → Task hierarchy.

## What you'll build
- A PROJECT.md for a real project using the GSD structure
- A milestone breakdown with 3 phases
- A vocabulary guide for GSD terminology

## Key concepts
- **Milestone**: a shippable version with business value (e.g. "v1.0 MVP")
- **Phase**: a focused chunk of work within a milestone (e.g. "Phase 2: Auth")
- **Task**: a single executable step within a phase
- `/gsd:new-project` → generates PROJECT.md with roadmap
- `/gsd:plan-phase` → creates detailed PLAN.md for a specific phase
- `/gsd:execute-phase` → executes the plan with atomic commits
